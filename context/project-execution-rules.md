# SYNTRA CORE — Project Execution Rules

## Objetivo

Claude Code debe operar como un equipo de ingeniería premium.

Todo desarrollo debe priorizar:

* claridad
* estabilidad
* escalabilidad
* consistencia
* performance
* experiencia premium

---

# Filosofía General

El proyecto debe construirse de forma:

* modular
* incremental
* organizada
* documentada
* mantenible

Nunca generar caos estructural.

---

# Reglas de Ejecución

## Construcción progresiva

Implementar el proyecto por etapas claras.

---

## Orden recomendado

1. estructura base
2. layout principal
3. design system
4. navegación
5. secciones core
6. componentes reutilizables
7. motion system
8. responsive
9. optimización
10. SEO
11. testing final

---

# Reglas de Modificación

Antes de modificar archivos importantes:

* analizar arquitectura existente
* respetar estructura actual
* evitar romper componentes
* mantener consistencia visual

---

# Evitar

* refactors innecesarios
* cambios masivos sin motivo
* duplicación
* sobreingeniería

---

# Reglas de Componentización

Todo componente debe ser:

* reutilizable
* desacoplado
* limpio
* escalable

---

# Reglas de UI

Toda nueva UI debe respetar:

* design-rules.md
* ui-patterns.md
* motion-rules.md
* brand-guide.md

---

# Reglas de Motion

Las animaciones deben ser:

* suaves
* mínimas
* premium
* performantes

Nunca exageradas.

---

# Reglas de Código

Todo código debe ser:

* limpio
* tipado
* organizado
* mantenible
* consistente

---

# TypeScript

Usar TypeScript estricto.

Evitar:

* any
* lógica ambigua
* tipados inseguros

---

# Performance Rules

Priorizar siempre:

* Lighthouse +95
* velocidad de carga
* responsive fluido
* optimización de imágenes
* mínimo layout shift

---

# SEO Rules

Implementar:

* metadata correcta
* semantic HTML
* headings organizados
* performance SEO-friendly

---

# Responsive Rules

Mobile-first obligatorio.

Toda sección debe funcionar perfectamente en:

* mobile
* tablet
* desktop

---

# UX Philosophy

La experiencia debe sentirse:

* premium
* moderna
* tecnológica
* inteligente
* fluida

---

# Documentación

Documentar:

* decisiones importantes
* estructuras complejas
* lógica crítica
* integraciones

---

# Reglas para Dependencias

Evitar instalar librerías innecesarias.

Priorizar:

* stack oficial
* librerías modernas
* herramientas performantes

---

# Reglas de Arquitectura

Respetar:

* frontend-architecture.md
* backend-architecture.md
* development-rules.md

---

# Testing Philosophy

Validar constantemente:

* responsive
* errores
* accesibilidad
* performance
* consistencia visual

---

# Filosofía Final

Claude Code debe construir SYNTRA CORE como:

* un producto tecnológico premium
* un ecosistema AI-native
* una plataforma moderna
* infraestructura digital escalable

Nunca como una web corporativa genérica.
