# SYNTRA CORE — UI Patterns

## Objetivo

El sistema UI de SYNTRA CORE debe sentirse:

* premium
* moderno
* tecnológico
* minimalista
* limpio
* escalable
* sofisticado

La interfaz debe parecer un producto SaaS moderno de alta calidad.

Nunca una web corporativa tradicional.

---

# Inspiraciones Oficiales

Referencias principales:

* Linear
* Vercel
* Stripe
* Framer
* Raycast
* Supabase

---

# Filosofía del Sistema UI

La interfaz debe priorizar:

* claridad visual
* jerarquía limpia
* espacio negativo
* simplicidad
* consistencia
* legibilidad
* refinamiento visual

---

# Layout Rules

## Containers

Usar contenedores amplios.

La UI debe respirar.

---

## Espaciado

Priorizar espaciado generoso entre:

* secciones
* componentes
* bloques de contenido
* cards
* elementos interactivos

Nunca saturar visualmente la interfaz.

---

## Grid System

Preferir:

* grids simétricos
* alineaciones limpias
* layouts modulares
* estructura consistente

---

# Border Radius

Border radius oficial:

* rounded-2xl
* rounded-3xl para elementos premium

Evitar bordes demasiado pequeños o cuadrados.

---

# Cards

Las cards deben sentirse:

* modernas
* limpias
* premium
* flotantes
* refinadas

---

## Características recomendadas

* fondos oscuros suaves
* bordes sutiles
* transparencia moderada
* blur ligero
* sombras suaves
* separación clara

---

## Evitar

* sombras agresivas
* bordes pesados
* estilos antiguos
* demasiada información visual

---

# Buttons

Los botones deben sentirse:

* sólidos
* modernos
* premium
* rápidos
* táctiles

---

## Estilo recomendado

* alto contraste
* padding amplio
* hover elegante
* transiciones suaves
* glow sutil

---

## Evitar

* botones demasiado pequeños
* gradients exagerados
* bordes anticuados
* estilos recargados

---

# Typography Rules

La tipografía debe sentirse:

* limpia
* moderna
* tecnológica
* sofisticada

---

## Jerarquía

Usar:

* títulos grandes
* subtítulos claros
* textos cortos
* espaciado consistente

---

## Evitar

* párrafos enormes
* exceso de texto
* bloques densos

---

# Iconography

Los íconos deben ser:

* minimalistas
* modernos
* consistentes
* lineales

Preferir Lucide Icons.

---

# Glassmorphism

Glassmorphism debe utilizarse con moderación.

---

## Recomendado

* blur suave
* transparencia ligera
* overlays elegantes

---

## Evitar

* blur excesivo
* fondos demasiado transparentes
* ruido visual

---

# Shadows

Las sombras deben ser:

* suaves
* profundas
* elegantes

Nunca agresivas.

---

# Color Usage

La UI debe priorizar:

* dark mode
* contraste limpio
* acentos tecnológicos
* paleta consistente

---

## Evitar

* demasiados colores
* saturación excesiva
* gradients fuertes
* inconsistencias visuales

---

# Responsive Philosophy

El diseño debe ser:

* mobile-first
* ultra responsive
* adaptable
* fluido

La experiencia mobile debe sentirse premium.

---

# Section Design

Cada sección debe sentirse:

* modular
* limpia
* espaciosa
* enfocada

---

## Recomendado

* separación clara
* ritmo visual consistente
* jerarquía fuerte

---

# Hero Sections

Los heroes deben transmitir:

* impacto visual
* sofisticación
* innovación
* identidad tecnológica

---

## Elementos recomendados

* headlines grandes
* motion elegante
* gradients suaves
* glow sutil
* background depth
* visual hierarchy clara

---

# UI Consistency

Todos los componentes deben seguir:

* spacing consistente
* misma lógica visual
* mismas reglas de interacción
* mismo lenguaje visual

---

# Performance Rules

La UI debe verse premium SIN comprometer:

* velocidad
* accesibilidad
* performance
* SEO

---

# Principio Final

La interfaz de SYNTRA CORE debe sentirse como:

* un producto tecnológico premium
* una plataforma moderna
* un ecosistema AI-native
* infraestructura digital avanzada

Nunca como una agencia tradicional.
