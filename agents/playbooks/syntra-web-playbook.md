# SYNTRA WEB PLAYBOOK — CORE SYSTEM

---

# 1. PROPÓSITO

## RELACIÓN CON WEB DELIVERY PIPELINE

Este documento es una guía operativa.

NO reemplaza al WEB DELIVERY PIPELINE.

La autoridad sobre estados, transiciones, validaciones y aprobaciones vive exclusivamente en:

👉 WEB DELIVERY PIPELINE

Este Playbook existe para explicar:

- cómo ejecutar el proceso
- cómo aplicar los frameworks
- cómo coordinar agentes

Ante cualquier conflicto:

WEB DELIVERY PIPELINE prevalece.

Este documento define el proceso oficial para diseñar, construir, validar y desplegar sitios web premium dentro de SYNTRA CORE.

Su objetivo es transformar una necesidad de negocio en una experiencia digital premium, consistente y diferenciada.

Este playbook conecta:

* ROLE-AUTHORITY-MAP
* WEB DELIVERY PIPELINE
* SYNTRA PREMIUM STANDARD

en un único flujo operativo.

---

# 2. PRINCIPIO FUNDAMENTAL

Una web premium no es una colección de secciones.

Es una experiencia diseñada intencionalmente.

El objetivo no es producir páginas.

El objetivo es construir experiencias memorables que generen confianza, diferenciación y conversión.

---

Toda iniciativa web debe seguir el siguiente recorrido:

Website Experience Auditor

Creative Director
↓

---

# 4. FASE 0 — DESCUBRIMIENTO

Objetivo:

Comprender el negocio antes de diseñar cualquier solución.

Preguntas obligatorias:

* ¿Qué vende la empresa?
* ¿Qué problema resuelve?
* ¿Quién es el cliente ideal?
* ¿Qué la diferencia de sus competidores?
* ¿Qué percepción quiere generar?
* ¿Qué no queremos parecer?
* ¿Qué debería recordar el usuario después de visitar la web?

Output:

* contexto de negocio
* posicionamiento
* objetivos de conversión
* restricciones

---

# 5. FASE 1 — AUDITORÍA

Responsable:

Website Experience Auditor

Objetivo:

Evaluar el estado actual y detectar oportunidades.

Herramientas obligatorias:

* SYNTRA PREMIUM STANDARD
* Auditoría vía Website Experience Auditor

Evaluar:

* diferenciación
* narrativa
* memorabilidad
* conversión
* deuda experiencial
* patrones commodity

Output:

* fortalezas
* debilidades
* oportunidades
* score de experiencia

---

# 6. FASE 2 — DIRECCIÓN CREATIVA

Responsable:

Creative Director

Objetivo:

Definir la visión creativa del proyecto.

Determinar:

* percepción deseada
* emoción dominante
* tono visual
* momento diferencial
* principios creativos

Preguntas obligatorias:

* ¿Qué debe sentir el usuario?
* ¿Qué debe recordar?
* ¿Qué hace única esta experiencia?

Output:

Creative Direction Brief

---

# 7. FASE 3 — DISEÑO DE EXPERIENCIA

Responsable:

Product Experience Designer

Objetivo:

Diseñar el recorrido completo del usuario.

Definir:

* narrativa
* ritmo
* jerarquía de información
* momentos de confianza
* momentos de identificación
* momentos diferenciales

Preguntas obligatorias:

* ¿Por qué sigue bajando el usuario?
* ¿Qué emoción genera cada sección?
* ¿Qué podría eliminarse?

Output:

Experience Architecture

---

# 8. FASE 4 — DISEÑO DE INTERFAZ

Responsable:

UI/UX Designer

Objetivo:

Materializar la experiencia en interfaz.

Definir:

* layouts
* composición
* jerarquía visual
* responsive
* motion
* patrones visuales

Debe respetar:

* Creative Direction Brief
* Experience Architecture
* Design System

Output:

UI Specification

---

# 9. FASE 5 — VALIDACIÓN DEL SISTEMA

Responsable:

Design System Guardian

Objetivo:

Garantizar consistencia visual global.

Validar:

* componentes
* design tokens
* estados UI
* reutilización
* coherencia visual

Output:

Design Validation

---

# 10. FASE 6 — IMPLEMENTACIÓN

Responsable:

Frontend Engineer

Objetivo:

Implementar exactamente lo aprobado.

Debe respetar:

* UI Specification
* Design Validation
* Frontend Delivery Protocol

Debe reportar:

* deuda técnica
* deuda visual
* deuda UX

Output:

Implementación funcional

---

# 11. FASE 7 — QA Y VALIDACIÓN FINAL

Responsable:

Web QA & Performance Guard

Objetivo:

Validar calidad técnica y experiencial.

Evaluar:

* funcionalidad
* performance
* responsive
* consistencia visual
* diferenciación
* narrativa
* percepción premium

Herramientas obligatorias:

* SYNTRA PREMIUM STANDARD
* Auditoría vía Website Experience Auditor

Output:

APPROVED
REQUIERE CORRECCIÓN
BLOQUEADO

# 12. CHECKLIST FINAL SYNTRA

Antes de aprobar una web deben responderse:

### Diferenciación

¿Qué no podría existir en cualquier agencia?

---

### Memorabilidad

¿Qué recordará el usuario?

---

### Conversión

¿Qué genera confianza?

---

### Narrativa

¿Existe progresión clara?

---

### Redundancia

¿Qué podría eliminarse?

---

### Momento Diferencial

¿Cuál es el elemento más distintivo de la experiencia?

---

### Calidad Premium

¿La experiencia se siente premium por claridad o por efectos?

Debe ser por claridad.

---

# 13. REGLA DE COMMODITY

Si una sección podría existir en cualquier competidor:

debe cuestionarse.

Si una experiencia no tiene un momento diferencial:

debe evolucionarse.

---

# 14. DEFINICIÓN DE ÉXITO

Una web SYNTRA es exitosa cuando:

* comunica valor rápidamente
* genera confianza
* posee al menos un momento diferencial
* evita patrones commodity
* mantiene coherencia narrativa
* deja una impresión memorable

---

# 15. PRINCIPIO FINAL

Las webs premium no se construyen agregando elementos.

Se construyen eliminando todo aquello que no aporta valor.

Si una decisión no mejora:

* claridad
* diferenciación
* confianza
* memorabilidad

no pertenece a la experiencia.
